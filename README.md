# Endometriosis Detection using Data Mining Techniques

A Streamlit app that predicts endometriosis risk from patient symptom/clinical
data, backed by a classification model trained and tuned across 5 algorithms.

## Files

| File | Purpose |
|---|---|
| `generate_data.py` | Creates a synthetic dataset (`endometriosis_data.csv`). **Replace with your real dataset** — just keep the target column named `diagnosis` (1/0). |
| `train_model.py` | Full pipeline: cleaning → scaling → class balancing → trains Logistic Regression, Decision Tree, Random Forest, SVM, KNN → tunes each with `GridSearchCV` + 5-fold cross-validation → picks the best model by Recall → saves `best_model.pkl`, `scaler.pkl`, `feature_list.json`, `model_comparison.json`. |
| `app.py` | The Streamlit UI — real-time prediction + sidebar model comparison table. |
| `requirements.txt` | Python dependencies. |

## Run Locally

```bash
# 1. Create a virtual environment (recommended)
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Generate data (skip this and drop in your real CSV instead, if you have one)
python generate_data.py

# 4. Train the models
python train_model.py

# 5. Launch the app
streamlit run app.py
```

Streamlit will open automatically at `http://localhost:8501`.

## Using Your Real Dataset

Replace `endometriosis_data.csv` with your actual data. Requirements:
- Target column must be named `diagnosis` (1 = endometriosis, 0 = not).
- All other columns are treated as numeric/binary features automatically.
- If your feature names differ from the synthetic ones, no problem —
  `train_model.py` picks up whatever columns are in the CSV, and
  `feature_list.json` keeps `app.py` in sync automatically. You'll just want
  to update the input widgets in `app.py`'s "Input form" section to match
  your real feature names/ranges.

Then re-run steps 4 and 5 above.

## Deploying (Streamlit Community Cloud — free)

1. Push this folder to a GitHub repo (include the `.pkl`/`.json` files
   generated by `train_model.py`, or generate them on the server — see step 3 below).
2. Go to [share.streamlit.io](https://share.streamlit.io), sign in with GitHub.
3. Click **"New app"**, select your repo/branch, and set the main file to `app.py`.
   - If you don't want to commit the `.pkl` files, add a `packages.txt`/build
     step, or simply run `generate_data.py` + `train_model.py` once locally
     and commit the resulting artifacts — simplest for a demo/portfolio app.
4. Click **Deploy**. Streamlit Cloud installs `requirements.txt` automatically
   and gives you a public URL you can share in your resume/portfolio.

## Model Selection Rationale

The best model is chosen by **Recall**, not raw accuracy — in a diagnostic
screening context, a false negative (missing a true endometriosis case) is
more costly than a false positive (flagging a healthy patient for extra
review). `model_comparison.json` keeps every algorithm's full metrics
(Accuracy, Precision, Recall, F1, ROC-AUC) so you can justify the choice in
an interview.
